微信JS SDK

需引入微信JSSDK
此外，nativeShare是一个可以通过javascript直接调用原生分享的工具, 只有少数几个浏览器可跨应用调用微信的API，由于这个支持度比较少(UC浏览器和QQ浏览器)，就不讲了demo

HTML5 之 pushState

功效

  1. 对于单页局部更新数据,整体页面无刷—ajax
  2. 到要求地址栏url跟随变动—pushState replaceState
  3. 切浏览器前进后退按钮有效—popstate onpopstate
  4. pjax

    api 形式

    history.pushState(state, title, href);
    history.replaceState—-参数一样
    window.addEventListener(‘popstate’, function (e) {})

使用

  1. pushState 可将设置当前地址框url ,title, 并将当前用以渲染的数据塞入history供前进后退时重新render时使用
  2. replaceState 同上，不同的是是替换信息，可用作初次渲染使用，从而可回退到首屏，并作为当前窗口的历史第一屏
  3. 对于前进后退按钮，通过绑定popstate事件，获取当时塞入的数据重新渲染即可

无能为力的事

1、设置的url无法跨域
2、存在history中的数据state, 其各个字段的值只能是字符串，不能时对象，state本身属性可多个（任意）

浏览器兼容性

Feature Chrome Firefox (Gecko) Internet Explorer Opera Safari
replaceState, pushState 5 4.0 (2.0) 10 11.50 5.0
history.state 18 4.0(2.0) 10 11.50 6.0

require 笔记

官方文档

代码组织结构

在相应的html中通过 引入代码.
require.js是所有页面js的入口， data-main对应的才是该页面真实的js入口
js目录结构中，page中为每个页面对应的入口，每个页面相当于是一个应用，这些入口js是应用级的，不可复用
module为针对每个功能开发的模块，requirejs的模板就是鼓励代码模块化
lib为按需载入的各个第三方库，如jqurey,等
原则：尽量模块化，高复用，全局变量要尽量少

page 入口文件

requirejs config

config详解

  • 所有模块，依赖都默认为是js,无需加上.js后缀
  • config需在入口处，保证先执行，也可写在html中
  • paths 映射不直接放置在baseUrl下的模块名，一般相对路径，之后，可用映射名代替这一模块，一般用以映射lib库, 映射对象也可是绝对路径，或者完整URL(with https?)
  • shim 可将非AMD规范的依赖规范化，之后，每项可直接作为依赖直接使用了, 。每项包含3个参数：
  1. exports可将这一依赖注入到全局中，作为全局变量
  2. deps:[该依赖的依赖列表]，其依赖在配置时并未加载，只有当改依赖需要被载入时
  3. init: construct function + 解决变量冲突的问题(由于不符合AMD规范的库可能使用相同的全局变量，从而引起冲突，一般是通过第三方依赖的noConflict方法实现)
  • map 用以进行版本映射，以在不同页面中使用不同的依赖版本，一般用不着，谁没事弄几个版本的代码一起用啊
  • config 应用级的参数传递，用以向模块(modules中的文件)传递参数，应用级，即不同的页面调用相同的子模块实现不同的效果，向子模块传递参数，每一项是一个模块，以及该模块需使用的参数,如，在 模块getMore中，可使用module.config().total获取该参数
  • 其他，packages(引入包，其住模块为index.js)，urlArgs, scriptType…
